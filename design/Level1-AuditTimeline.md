### 2.2 选择策略

**Easy 难度选择原则**：
1. 优先选择**大标签**（视觉上最显眼，最容易扫描）
2. 只选择出现 **1 次** 的关键字（复制后恰好 2 个，玩家容易理解）
3. 避免选择**相似标签**（如 `<div>` 和 `<span>` 同时作为配对，会增加干扰）
4. 植入位置要"看起来合理"（像复制粘贴错误）

**最终选择 8 对**：
- 全部选择**只出现1次的大标签**
- 理由：第一关要让玩家快速理解"找重复"的核心玩法，大标签最显眼

## 6. 难度曲线分析

### 扫描难度梯度

| 配对编号 | 元素 | 扫描难度 | 原因 |
|---------|------|---------|------|
| 2 | `</template>` | ⭐ | 两个连续出现，最显眼 |
| 1 | `<template>` | ⭐ | 一个在开头，一个在中间 |
| 3 | `<el-timeline>` | ⭐⭐ | 上下分布，标签大 |
| 4 | `</el-timeline>` | ⭐⭐ | 上下分布，标签大 |
| 5 | `<el-timeline-item>` | ⭐⭐⭐ | 和 `<el-timeline>` 相似 |
| 6 | `</el-timeline-item>` | ⭐⭐⭐ | 和 `</el-timeline>` 相似 |
| 7 | `<el-input>` | ⭐⭐⭐⭐ | 一个在 template，一个在 style 前 |
| 8 | `</el-input>` | ⭐⭐⭐⭐⭐ | 一个在 template，一个在 style 内部 |

**设计意图**：
- 前2对让玩家快速建立信心（最简单）
- 中间4对增加扫描难度（相似标签干扰）
- 最后2对作为挑战（跨区域分布，最难找）
- 整体难度呈递增曲线，符合心流理论

## 7. 错误点击场景

### 场景 1：混淆相似标签
- 玩家选中 `<el-timeline>`（第7行）
- 然后点击 `<el-timeline-item>`（第8行）
- 结果：不匹配，红色下划线闪烁，扣10分，打断连击
- 反馈："标签不匹配！继续扫描..."

### 场景 2：混淆原始和植入（不存在）
- 本设计所有配对元素都恰好出现2次，没有"多余原始"
- 玩家只要选中两个相同的元素，就一定匹配

### 场景 3：已消除元素误点击
- 玩家点击已消除的灰色 `<el-timeline>`
- 结果：无反应（或提示"已标记删除"）

## 9. 视觉设计规范

### 关键字渲染
- 原始关键字：正常语法高亮（如 `<el-timeline>` 显示为蓝色）

- 消除状态：`.eliminated { color: #6a737d; text-decoration: line-through; opacity: 0.6; }`（灰色删除线）
- 错误状态：`.error { border-bottom: 2px solid #f14c4c; animation: shake 0.3s; }`

### 代码区域
- 字体：Consolas / 'Courier New' / monospace
- 字号：14px
- 行高：1.5
- 行号：左侧灰色 `#6e7681`，右对齐
- 背景：VS Code 暗色主题 `#1e1e1e`
- 语法高亮：模拟 Vue 文件高亮（HTML 标签蓝色、属性橙色、字符串绿色、JS 关键字紫色等）
