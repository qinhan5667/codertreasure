# CoderTreasure: Code Refactor — 游戏设计文档 (GDD) v1.0

## 1. 游戏概述

**游戏名称**：CoderTreasure: Code Refactor  
**类型**：伪装型配对消除（隐藏于代码编辑器中）  
**目标平台**：VS Code 扩展插件  
**目标玩家**：开发者、代码审查员、"摸鱼"需求者  
**核心体验**：玩家在审查真实代码的过程中，找出并消除"重复植入的冗余关键字"，完成代码重构。

**设计支柱**：
1. **极致伪装**：从任何角度看都是一名工程师在审查代码。消除是"删除冗余"，得分是"重构进度"。
2. **认知扫描**：核心乐趣 = 在复杂代码中快速扫描、定位重复关键字。不是反应速度，而是观察力和模式识别。
3. **渐进复杂**：从简单标签到复杂属性，逐步增加认知负荷。

---

## 2. 核心玩法循环

### Moment-to-Moment（0-30秒）
- **Action**：玩家眼睛扫描代码，寻找"可疑的重复关键字"。
- **Feedback**：鼠标悬停关键字时，出现微妙的下划线（像 VS Code 的 hover 提示）。
- **Reward**：找到重复关键字时，产生"发现 bug"的内在满足感。

### 消除动作（1-3秒）
- **Action**：点击第一个关键字 → 蓝色背景高亮（像选中代码）。点击第二个相同关键字 → 两个同时变为灰色/删除线（像标记为删除）。
- **Feedback**：成功配对时，短暂绿色闪光；错误配对时，微妙红色下划线（像 ESLint 报错）。
- **Reward**：+100分，连续消除有连击倍数。

### Session Loop（5-8分钟/关）
- **Goal**：消除当前代码文件中所有重复植入的关键字对。
- **Tension**：随着消除进行，剩余关键字越来越少，需要更仔细地扫描。
- **Resolution**：所有对消除 → 代码"审查完成"，进入下一关（新的代码文件）。

### Long-Term Loop（多关）
- **Progression**：解锁新语言（Vue → React → TypeScript → Python → Go...）。
- **Retention Hook**：每日挑战（随机代码文件），排行榜（最快审查时间）。

## 3. 第一关设计：AuditTimeline.vue

### 3.1 关卡概述
- **文件**：`AuditTimeline.vue`（Element Plus + Vue 3 SFC）
- **难度**：Easy（标签新手）
- **配对数**：8对
- **目标时间**：2-3分钟
- **伪装场景**："正在审查同事提交的 Vue 组件代码，寻找重复标签和冗余指令"

### 3.2 配对元素选择

| 编号 | 配对元素 | 原始出现次数 | 操作 | 植入位置 | 难度 |
|------|---------|-------------|------|---------|------|
| 1 | `<template>` | 1次（第1行） | 复制1次 | 插入到 `<script setup>` 前面 | Easy |
| 2 | `</template>` | 1次（最后） | 复制1次 | 插入到 `<script>` 前面 | Easy |
| 3 | `<el-timeline>` | 1次（第7行） | 复制1次 | 插入到 `<div class="audit-opinion-content">` 内部 | Easy |
| 4 | `</el-timeline>` | 1次（第17行） | 复制1次 | 插入到 `<div class="input-area">` 前面 | Easy |
| 5 | `<el-timeline-item>` | 1次（第8行） | 复制1次 | 插入到 `<el-timeline>` 内部 | Easy |
| 6 | `</el-timeline-item>` | 1次（第16行） | 复制1次 | 插入到 `<el-timeline>` 和 `</el-timeline>` 之间 | Easy |
| 7 | `<el-input>` | 1次（第19行） | 复制1次 | 插入到 `<div class="input-area">` 后面 | Easy |
| 8 | `</el-input>` | 1次（第21行） | 复制1次 | 插入到 `<div class="input-area">` 和 `</div>` 之间 | Easy |

**植入策略**：
- 植入位置看起来像一个"手滑重复"或"复制粘贴错误"。
- 植入的关键字在语法上可能不完全合理，但视觉上足够像真实代码。
- 原始关键字和植入关键字没有任何视觉区别，玩家必须靠扫描发现。

### 3.3 界面伪装设计

| 游戏元素 | 伪装方案 | 实现细节 |
|---------|---------|---------|
| **剩余对数** | 状态栏 `Ln 12, P: 5` | 后两位显示剩余对数，伪装成行号/列号 |
| **分数** | 不显示分数 | 或伪装成"Code Issues: 0"（消除完变成0） |
| **Hint 按钮** | VS Code Search 图标（🔍） | 点击像"查找下一个引用" |
| **New Game** | "Open File" 或切换 Git Branch | 看起来像打开新文件审查 |
| **消除动画** | 关键字灰色 + 删除线 | 像 git diff 中被删除的代码 |
| **点击高亮** | 蓝色背景 | VS Code 默认选中文本颜色 |
| **错误提示** | 微妙震动 + 红色下划线 | 像 ESLint 报错 |
| **标题栏** | `AuditTimeline.vue - Project - Visual Studio Code` | 完全正常 |
| **侧边栏** | Explorer 文件树 | 显示当前文件和关联文件 |

### 3.4 消除效果

- **成功配对**：两个关键字变为 `color: #6a737d`（灰色） + `text-decoration: line-through`（删除线），保留在代码中。
- **错误配对**：点击的第二个关键字闪烁红色下划线（0.5秒），扣10分，打断连击。
- **全部消除**：状态栏显示"Code Issues: 0"，提示"审查完成"，可进入下一关。

## 4. 难度分级系统

### Easy（初级审查员）
- **代码长度**：20-40 行
- **关键字类型**：大标签（`<template>`、`<el-timeline>`、`<el-input>`）
- **配对数**：6-8 对
- **干扰度**：0（只有大标签，没有相似关键字）
- **目标时间**：2-3 分钟
- **得分倍率**：1x

### Medium（中级审查员）
- **代码长度**：40-80 行
- **关键字类型**：标签 + 属性（`v-for`、`v-if`、`class="..."`、`v-model`）
- **配对数**：8-12 对
- **干扰度**：中（有 `</div>` 和 `</span>` 等相似标签，但只有部分形成配对）
- **目标时间**：3-5 分钟
- **得分倍率**：1.5x

### Hard（高级审查员）
- **代码长度**：80-120 行
- **关键字类型**：JS 表达式、变量名（`defineProps`、`computed`、`getImage`、`item.status`）
- **配对数**：12-18 对
- **干扰度**：高（大量相似关键字，如 `item`、`log`、`data` 等）
- **目标时间**：5-8 分钟
- **得分倍率**：2x

## 5. 得分系统

| 事件 | 得分 | 备注 |
|------|------|------|
| 消除一对 | +100 | 基础分 |
| 连续消除（无错误） | ×1.5, ×2, ×2.5... | 连击倍数 |
| 错误点击（不匹配） | -10 | 打断连击 |
| 使用提示 | -50 | 提示高亮下一对 |
| 时间奖励 | 剩余秒数 × 5 | 快速完成奖励 |
| 全关完成 | +500 | 审查完成奖励 |

## 6. 代码文件选择策略

### 第一关：AuditTimeline.vue（Vue/Element Plus）
- 选择原因：用户提供的真实代码，结构清晰，标签类丰富
- 配对元素：8 对标签类
- 伪装度：★★★★☆（看起来完全是一个 Vue SFC 文件）

### 第二关（建议）：React 组件
- 选择原因：引入 JSX 语法，增加干扰项
- 配对元素：`<div>` / `<Component>` / `useState` / `useEffect` 等
- 难度提升：JSX 和 HTML 标签混合，更难区分

### 第三关（建议）：TypeScript 类型定义
- 选择原因：纯类型代码，没有标签
- 配对元素：`interface` / `type` / `extends` / `readonly` 等
- 难度提升：关键字更抽象，视觉相似度高

## 7. 技术实现要点

### 7.1 代码渲染
- 使用 `<pre>` + `<code>` 标签渲染完整代码文件
- 语法高亮：自定义 CSS 模拟 VS Code 主题（Dark+/Light+）
- 行号：使用 CSS 计数器或独立 `<span>` 实现
- 可点击关键字：用 `<span class="keyword" data-keyword="el-timeline">` 包裹

### 7.2 关键字植入算法
1. 解析代码文件，统计所有关键字出现次数
2. 筛选出现 1 次的关键字（作为候选）
3. 随机选择 N 个候选关键字
4. 在代码中随机位置（不破坏语法结构）插入副本
5. 确保植入后该关键字恰好出现 2 次

### 7.3 交互逻辑
- 点击 `<span class="keyword">` → 添加 `.selected` 类（蓝色背景）
- 点击第二个 → 检查 `data-keyword` 是否相同
- 相同 → 添加 `.eliminated`（灰色 + 删除线）
- 不同 → 第二个闪烁红色下划线，重置选择
- 全部消除 → 触发胜利条件

### 7.4 伪装状态栏
- 左侧：分支名（如 `main`）+ 错误数（`Code Issues: 5`）
- 右侧：行号/列号（`Ln 12, Col 15`）+ 编码（`UTF-8`）+ 语言模式（`Vue`）+ Prettier 标识
- 剩余对数伪装：`Code Issues: 5` 随消除递减

## 8. 玩家体验流程

### 首次打开
1. 界面显示 VS Code 编辑器，加载 `AuditTimeline.vue`
2. 玩家以为只是一个代码预览工具
3. 鼠标悬停代码关键字，发现下划线效果
4. 点击一个 `<el-timeline>`，发现蓝色高亮
5. 点击另一个 `<el-timeline>`，发现灰色删除线 + "Code Issues: 7" → "Code Issues: 6"
6. 恍然大悟：这是一个游戏！

### 进阶体验
1. 加快扫描速度，建立连击
2. 尝试不使用提示，追求高分
3. 解锁 Medium 难度，面对更复杂的代码
4. 解锁 Hard 难度，挑战纯 JS/TS 代码审查
5. 每日挑战：随机 GitHub 代码文件

## 9. 风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 植入关键字破坏代码语法 | 伪装性降低 | 植入位置选择"看起来合理"的位置（如多余的闭合标签） |
| 玩家误操作真实代码 | 无影响 | 游戏只读，不修改实际文件 |
| 代码太长导致滚动困难 | 中等 | 限制代码长度（Easy 40行，Hard 120行） |
| 关键字太小难以点击 | 中等 | 确保关键字 `<span>` 有最小 `padding` |
| 颜色相似难以区分 | 低 | 消除后灰色 + 删除线，与正常语法高亮明显不同 |
